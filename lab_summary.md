MCP is useful when an agent needs a standard way to access external systems such as files, Git repositories, APIs, or databases because the same MCP pattern can be reused across different tools and frameworks. Direct API integration can be simpler for one small, specific task because the code talks directly to one service, but it usually requires custom code for every new system. In this lab, MCP is the better fit for filesystem and Git because LangChain can load both tool sets through one consistent interface. Git is required and fails early if no real repository is configured. Trello is an optional live integration with no mock mode: when configured, it writes to real Trello resources; when not configured, it is skipped. Trello is implemented both ways for comparison: the third-party `@delorenj/mcp-server-trello` package exposes Trello actions as LangChain tools through MCP, while direct official REST API calls show the explicit HTTP implementation and reduce indirection.
