MCP is useful when an agent needs a standard way to access external systems such as files, Git repositories, APIs, or databases because the same MCP pattern can be reused across different tools and frameworks. Direct API integration can be simpler for one small, specific task because the code talks directly to one service, but it usually requires custom code for every new system. In this lab, MCP is the better fit because LangChain can load filesystem and Git tools through one consistent interface, while direct API calls would require separate Python logic for file operations and Git operations.
