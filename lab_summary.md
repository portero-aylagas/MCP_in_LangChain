MCP is useful when an agent needs a standard way to access external systems such as files, Git repositories, APIs, or databases because the same MCP pattern can be reused across different tools and frameworks. Direct API integration can be simpler for one small, specific task because the code talks directly to one service, but it usually requires custom code for every new system. In this lab, MCP is the better fit for filesystem and Git because LangChain can load both tool sets through one consistent interface. Trello is handled with direct official REST API calls instead of a third-party MCP server, which reduces supply-chain risk while still demonstrating an external API write.
